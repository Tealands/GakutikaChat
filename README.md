# Evidence-Based 自己アピールシート生成AI

## 概要（What）
本プロジェクトは、就職活動における「ガクチカ（学生時代に力を入れたこと）」を  
**文章生成ではなく、行動・成果・証拠を一体化した「自己アピールシート」**として作成するAI支援ツールです。

単にそれっぽい文章を作るのではなく、
- 実際の行動
- その成果
- 第三者が確認可能な証拠  
を前提に構造化・言語化することを目的としています。

---

## 背景（Why）
多くの学生が以下の課題を抱えています。

- ガクチカに書ける経験がないと思い込んでいる
- 実体験はあるが、どう書けば評価されるか分からない
- 書くと感想文・日記になってしまう

一方、企業側にも以下の課題があります。

- 文章が上手いだけの学生と中身のある学生の見極めが難しい
- ESと面接内容の乖離が大きい
- 書類選考のノイズが多い

本プロジェクトは、これら双方の課題を同時に解決することを目指しています。

---

## 特徴（Key Features）

### 1. 質問型AIによる構造化
AIが一問一答形式で質問を行い、回答を蓄積。
最終的に STAR / PREP 構造に再構成します。

- 嘘や脚色を前提としない設計
- 「何を書けばいいか分からない」状態を回避

---

### 2. ガクチカ“創出”支援（行動提案）
「書ける経験がない」学生に対し、
- サークル活動
- 授業のグループワーク
- 学内活動  

などから、**短期・小規模・証明可能な行動**を提案します。

※ 架空エピソードの生成は行いません。

---

### 3. Evidence（証拠）前提の設計
自己アピールシートには以下を紐づけ可能です。

- 写真（作業中・成果物など）
- URL（資料・記事・成果物）
- GitHub リポジトリ
- スクリーンショット

文章と証拠の整合性チェックも行います。

---

### 4. IT学生向け GitHub 活用
GitHub リポジトリを証拠として利用可能。

面接官向けに、以下構成の README 要約を提案します。

- 解決した課題
- 自分の担当範囲
- 技術的工夫
- 失敗と改善
- 再現性のある学び

コードが読めない面接官でも評価しやすい形を想定しています。

---

## 成果物（Output）

本ツールの最終アウトプットは以下です。

### 自己アピールシート
- Experience Summary（要約）
- Problem & Action
- Outcome
- Evidence（証拠）
- 再現性・学び

ES提出・面接双方で利用可能な1枚資料を想定しています。

---

## 技術スタック（想定）

- Frontend: Next.js / React
- Backend: API Routes
- AI: 既存LLM API（学習・追加データ不要）
- Storage: Supabase / Firebase / S3互換
- Auth: MVPでは省略または簡易認証

※ 本プロジェクトは「AIモデルを学習する」ことを目的としていません。

---

## なぜデータなしで成立するのか
本プロジェクトでAIが担うのは以下です。

- 質問の提示
- 回答の整理
- 構造化（STARなど）
- 整合性チェック

いずれも **既存LLMの推論能力 + プロンプト設計**で実現可能であり、
独自学習データは不要です。

---

## 想定利用者

- 就活初期〜中盤の大学生
- ガクチカの言語化に悩む学生
- IT・非ITを問わず利用可能

---

## 将来的な展開

- 企業向け閲覧UI
- 面接官向け深掘り質問自動生成
- 利用ログを活用した質問最適化
- 就活支援企業・大学キャリアセンターへの提供

---

## 目的（Vision）
本プロジェクトは、
「文章が上手い学生」ではなく  
**「行動を説明できる学生」が正当に評価される就活環境**を目指します。

ES選考を、文章審査から行動理解へ。
